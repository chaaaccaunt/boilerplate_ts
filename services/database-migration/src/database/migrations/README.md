# Миграции базы данных

SQL-файлы миграций должны лежать в этой директории и выполняться в лексикографическом порядке.
Если для dialect нужен отдельный SQL-синтаксис, миграции размещаются в поддиректории с именем Sequelize dialect.
Например, PostgreSQL-миграции лежат в `postgres`.
Если dialect-specific директория существует, migration service выполняет ее вместо корневого набора SQL-файлов.

Рекомендуемый формат имени:

```text
YYYYMMDDHHMMSS_описание_миграции.sql
```

Каждая успешно выполненная миграция записывается в таблицу `database_migrations`.
