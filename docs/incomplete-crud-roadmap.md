# Неполные CRUD-сценарии

На текущий момент известные неполные CRUD-зоны закрыты.

## roles

Статус: полный CRUD реализован.

- `create` — реализовано;
- `read/list` — реализовано;
- `update` — реализовано для пользовательских ролей;
- `delete` — реализовано для пользовательских ролей, которые не назначены пользователям.

Системные роли `administrator` и `user` остаются обязательной infrastructure boilerplate и защищены от переименования и удаления.

## chat messages

Статус: полный CRUD реализован.

- `create` — реализовано через `chat:message:send`;
- `read/list` — реализовано;
- `update` — реализовано для собственного сообщения;
- `delete` — реализовано для собственного сообщения.

Изменения и удаления отправляют realtime notifications `chat:message:updated` и `chat:message:deleted` другим участникам комнаты.

## stored files

Статус: полный пользовательский CRUD реализован.

- `create/upload` — реализовано;
- `read/list/download/view/preview` — реализовано;
- `update metadata` — реализовано через обновление описания;
- `delete` — реализовано как soft delete собственного файла, если файл не прикреплен к сообщению.

Пользовательское удаление не удаляет физический файл с диска. Физическая очистка файлов должна оставаться отдельным maintenance/admin-сценарием, чтобы не ломать историю сообщений и возможные будущие аудиторские сценарии.

## chat message files

Статус: CRUD связи реализован в рамках lifecycle сообщения.

- `create relation` — реализовано при отправке или редактировании сообщения с файлами;
- `read relation` — реализовано при загрузке сообщений;
- `update` — реализовано через замену набора вложений при редактировании сообщения;
- `delete` — реализовано как удаление отдельного вложения из собственного сообщения.
